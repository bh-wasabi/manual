#### document
- tipo de documento a generar el EDI

#### service
- servicio específico

#### scope
- se define la sección del documento a usar como base

#### condition1
- se pueden establecer condiciones para su ejecución

#### condition2
- se pueden establecer condiciones para su ejecución

#### condition3
- se pueden establecer condiciones para su ejecución

#### author
- se puede indicar el autor de este cambio

#### ignore (booleano)
- es posible comentar los renglones del metadata usando esta columna, tambien se recomienda poner el color de las letras en gris para que sea más visual lo comentado.

#### include
- lista de proyectos (separados por comas) que se desean incluir.

#### exclude
- lista de proyectos (separados por comas) que se desean excluir.

#### order
- orden en la ejecución del layout EDI

#### ediService
- nombre del servicio EDI que se esta generando el layout, debe corresponder con la configuracion de wasabiCfg

#### segment
- segmento del archivo EDI

#### seq
- sequencia dentro del segmento del archivo EDI

#### name
- nombre informativo de la columna que se esta generando

#### value
- valor base

#### value2
